# Аналіз портфоліо Дениса Загоровського

## Короткий висновок
Портфоліо справляє враження продуманої, мультимовної вітрини з сильним візуальним стилем та сучасним стеком (React, Vite, Tailwind, Framer Motion). Найбільші зони росту лежать у сфері практичної конверсії (контактна форма та довіра), деталізації кейсів, продуктивності на мобільних пристроях та глибшій SEO/аналітиці. Нижче — конкретні спостереження та покращення.

## Сильні сторони
- **Мультимовність і тема**: i18next з локалізаціями EN/PL/UA та ThemeProvider збереженням у `localStorage` (див. `src/context/ThemeContext.jsx`).
- **Анімації та мікровзаємодії**: послідовне використання Framer Motion, `AnimatePresence` у `App.jsx` для плавних переходів.
- **Компонентний підхід**: повторно використовувані блоки (`SectionTitle`, `ProjectCard`, `ProjectModal`) полегшують підтримку.
- **Оптимізований старт**: скрипт у `index.html` миттєво підтягує тему, що зменшує "flash of incorrect theme".

## Напрями для покращення

### 1. Контент та позиціонування
- **Бракує чітких доказів впливу**: у `projects.items.*.description` описано діяльність, але немає вимірюваних результатів (наприклад, "+20% швидше"), відгуків чи ролі команди. Додайте KPI, цитати менторів/клієнтів, посилання на пресу.
- **Відсутній CV/резюме**: додайте CTA на PDF або LinkedIn (можна використати окремий блок у `Header.jsx` чи `Footer.jsx`).
- **Немає блогу / новин**: варто показати останні виступи, статті, участь у хакатонах.

### 2. UX та інформаційна архітектура
- **Модальне вікно проєкту** (`ProjectModal.jsx`) закривається при кліку на фон — добре. Але на мобайлі текст займає багато місця; розгляньте full-screen слайд з жестом назад.
- **Навігація (Header)**: на десктопі кнопки з дуже широким `letter-spacing` (`tracking-[0.3em]`), що знижує читабельність, особливо українською. Зменшіть до `0.15em` або використайте звичайний регістр.
- **Дублювання CTA**: на головній лише два варіанти (проєкти/контакти). Додайте третій для "Завантажити резюме" чи "Приєднатися до ініціативи".
- **Відсутність breadcrumbs або індикатора сторінки**: при переході між сторінками добре показати, де користувач у структурі (особливо у `Header` на мобайлі).

### 3. Візуальна мова
- **Контраст у темній темі**: деякі вторинні тексти (наприклад, `text-slate-400` на темному фоні у `SectionTitle.jsx`) мають недостатній контраст за WCAG. Підніміть яскравість до `text-slate-300`.
- **Типографіка великими літерами**: часте використання uppercase + `tracking-[0.3em]` ускладнює читання українською/польською. Залиште uppercase лише для малих лейблів.
- **Герой-зображення**: бракує фото/ілюстрації автора. Додайте портрет або коротке відео для персоналізації.

### 4. Контактна форма та довіра
- **Неробоча відправка на GitHub Pages**: форма (`src/pages/Contact.jsx`) надсилає POST на `'/'` і розрахована на Netlify (`data-netlify="true"`). На GitHub Pages це поверне 404, а `fetch('/')` не обробить, тому користувач бачить "success", але лист не приходить. Потрібно:
  - або деплоїти на Netlify,
  - або підключити сторонній сервіс (Formspree, Getform) з API endpoint,
  - або використовувати власну serverless-функцію.
- **Відсутній стан `loading`**: `status` переходить з `idle` одразу до `success`/`error`, не показуючи, що запит обробляється. Додайте стан "Відправляю…" та дизейбл кнопки.
- **Немає альтернативних контактів**: у `Contact` показані соцмережі, але варто дублювати e-mail як посилання mailto.

### 5. Accessibility
- **ScrollRestoration** (`src/components/ScrollRestoration.jsx`) завжди викликає `window.scrollTo(..., behavior: 'smooth')`. Це ігнорує користувачів з `prefers-reduced-motion`. Додайте перевірку і для них вмикайте миттєвий скрол.
- **Описові `alt`**: зараз `ProjectCard` та `ProjectModal` використовують `alt={project.title}`. Замість назви вкажіть короткий зміст ілюстрації ("Скріншот панелі City Pulse з картою Львова").
- **Фокусні стилі**: деякі кнопки у модалці та соцлінках покладаються на кольоровий бордер. Додайте додатковий `outline-offset`, щоб фокус був помітним і на темному фоні.

### 6. Продуктивність та технічні деталі
- **Framer Motion**: велика кількість `motion.*` компонентів на сторінці збільшує бандл. Розгляньте lazy-експорт анімацій чи використання `motion` лише для контейнерів.
- **Ліниве завантаження зображень**: `loading="lazy"` використовується, але можна додати responsive `srcSet` або modern image formats (AVIF/WebP).
- **Кешування локалізацій**: ресурси зберігаються у бандлі. Для масштабування можна винести переклади в окремі JSON і завантажувати on-demand.
- **Відсутність PWA/manifest**: додайте `manifest.json`, іконки та офлайн-кешування (Vite plugin) для кращого мобільного досвіду.

### 7. SEO та соціальні мережі
- **Метадані на кожну сторінку**: `useDocumentTitle` оновлює title/description, але Open Graph та Twitter-теги зашиті в `index.html` і не міняються. Для кращого SEO створіть динамічні OG мета (можна через `react-helmet-async`).
- **Canonical + hreflang**: відсутні. Додайте `<link rel="canonical">` та `hreflang` для трьох мов.
- **Structured Data**: додайте JSON-LD (Person + Project) для Google Knowledge Graph.
- **Sitemap**: немає автоматично згенерованої sitemap.xml (можна зібрати через Vite плагін).

### 8. Аналітика та довіра
- **Відсутня аналітика**: підключіть Plausible/GA/Umami, щоб знати, які розділи переглядають рекрутери.
- **Соціальний доказ**: додайте секцію "Партнери" або "Медіа згадки".
- **Call-to-action для підписки**: форма email-дайджесту або Telegram-канал допоможе вибудувати аудиторію.

## Пріоритети впровадження

| Пріоритет | Що зробити | Навіщо |
|-----------|------------|-------|
| Високий | Полагодити контактну форму (endpoint + стан loading) | Головний канал конверсії зараз не працює |
| Високий | Додати конкретику в кейси та CTA на резюме | Рекрутери швидше зрозуміють ваш вплив |
| Середній | Зменшити letter-spacing, перевірити контраст у темній темі | Покращення читабельності та доступності |
| Середній | Оптимізувати ScrollRestoration й анімації для `prefers-reduced-motion` | Краще UX та відповідність WCAG |
| Низький | Динамічні OG теги, schema.org, sitemap | Довгострокове SEO |
| Низький | Додати аналітику та соціальний доказ | Вимірювання та зростання довіри |

## Додаткові ідеї
- Зробити розділ "Roadmap/Now" із переліком актуальних ініціатив.
- Додати інтерактивну карту/таймлайн активностей (Framer Motion вже використовується, можна зробити segment control).
- Вивести блок "Чим можу допомогти" з чіткими сервісами (консультації, розробка MVP, менторство).
- Записати коротке відео-привітання (30 секунд) та вставити на головну — це підвищує конверсію звернень.
